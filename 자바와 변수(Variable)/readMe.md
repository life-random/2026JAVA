# 02. 자바 변수(Variable)의 개념과 명명 규칙

## 1. 변수(Variable)란?
* **개념:** 데이터를 저장할 수 있는 **메모리 공간(상자)**입니다.
* **특징:** '변하는 수'라는 뜻대로, 저장된 데이터(값)는 프로그램 실행 중에 언제든지 다른 값으로 변경될 수 있습니다.
* **타입 지정:** 변수를 선언할 때는 저장할 데이터의 종류에 맞춰 자료형(Data Type)을 미리 지정해야 합니다.
  * 예: `String`(문자열), `int`(정수), `double`(실수), `boolean`(논리값) 등

---

## 2. 변수의 기본 사용 단계
1. **변수 선언:** 값을 담을 메모리 공간 확보 (예: `int age;`)
2. **값 대입 (초기화):** 변수에 실제 데이터를 저장 (예: `age = 30;` 또는 선언과 동시에 대입 `int age = 30;`)
3. **변수 접근/사용:** 변수 이름을 호출하여 저장된 값을 읽거나 수정 (예: `System.out.println(age);`)

---

## 3. 변수 선언 시 필수 규칙 (자바 문법 규칙)

1. **대소문자 구분:** 대소문자를 엄격하게 구분하며, 작성 길이에 제한이 없습니다.
   * `age`, `aGe`, `aGE`는 모두 서로 다른 변수로 인식됩니다.
2. **예약어 사용 불가:** 자바 언어 내부에서 미리 기능을 정해둔 키워드(예약어)는 변수 이름으로 쓸 수 없습니다.
   * 사용 불가 예시: `int`, `double`, `for`, `while`, `class` 등
3. **특수문자 제한:** 특수문자는 오직 **`_` (언더바)**와 **`$` (달러)** 표기만 허용됩니다.
   * 공백(띄어쓰기)이나 `*`, `-` 등의 다른 특수문자는 변수명에 사용할 수 없습니다.
4. **중복 선언 불가:** 같은 스코프(영역) 내에서는 동일한 이름의 변수를 중복해서 선언할 수 없습니다.

---

## 4. 변수 표기법 (Naming Convention)

코드를 읽기 쉽게 만들기 위해 상황에 따라 다음과 같은 표기법을 사용합니다.

| 표기법 | 설명 | 예시 | 비고 |
| :--- | :--- | :--- | :--- |
| **카멜 케이스 (Camel Case)** | 첫 글자는 소문자, 연결되는 단어의 첫 글자는 대문자 | `userName`, `ageBox` | **자바 변수/메서드 표준** |
| **스네이크 케이스 (Snake Case)** | 단어 사이를 언더바(`_`)로 연결 | `user_name`, `age_box` | 데이터베이스, Python 등 |
| **파스칼 케이스 (Pascal Case)** | 모든 단어의 첫 글자를 대문자로 표기 | `UserName`, `Variable` | **자바 클래스명 표준** |
| **케밥 케이스 (Kebab Case)** | 단어 사이를 하이픈(`-`)으로 연결 | `user-name`, `age-box` | HTML/CSS, URL 경로 등 |

---

## 5. 좋은 변수 이름 정하기
* 변수명은 해당 변수가 어떤 용도로 쓰이는지 **한눈에 의도를 알 수 있도록** 직관적으로 지어야 합니다.
